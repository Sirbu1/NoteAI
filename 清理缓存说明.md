# 解决 Gradle 编译错误的步骤

## 问题原因
Android SDK 35 和 36 版本较新，可能与当前的 JDK/JBR 版本不兼容，导致 `jlink.exe` 工具执行失败。

## 解决方案

### 步骤 1：将 compileSdk 改回 34（已完成）
✅ 已修改 `app/build.gradle`，将 `compileSdk` 从 35 改为 34

### 步骤 2：清理 Gradle 缓存（重要！）

#### 方法 A：在 Android Studio 中清理（推荐）

1. **清理项目缓存**
   - 点击菜单：`File` → `Invalidate Caches / Restart...`
   - 选择 `Invalidate and Restart`
   - 等待 Android Studio 重启

2. **清理项目**
   - 点击菜单：`Build` → `Clean Project`
   - 等待完成

3. **重新构建**
   - 点击菜单：`Build` → `Rebuild Project`
   - 等待完成

#### 方法 B：手动删除缓存文件夹（如果方法 A 不行）

1. **关闭 Android Studio**

2. **删除以下文件夹**：
   - `C:\Users\15591\.gradle\caches\transforms-3\` （整个文件夹）
   - 项目根目录下的 `.gradle` 文件夹（如果存在）
   - 项目根目录下的 `build` 文件夹（如果存在）
   - 项目根目录下 `app\build` 文件夹（如果存在）

3. **重新打开 Android Studio**

4. **同步项目**
   - 点击：`File` → `Sync Project with Gradle Files`

### 步骤 3：验证修复

1. 等待 Gradle 同步完成
2. 点击 `Build` → `Rebuild Project`
3. 如果看到 "BUILD SUCCESSFUL"，说明修复成功

## 为什么使用 SDK 34？

- ✅ SDK 34 (Android 14) 是稳定版本
- ✅ 兼容性更好，问题更少
- ✅ 对于学习项目完全够用
- ❌ SDK 35/36 是较新版本，可能存在兼容性问题

## 如果问题仍然存在

1. **检查 Android SDK 安装**
   - `File` → `Settings` → `Appearance & Behavior` → `System Settings` → `Android SDK`
   - 确保已安装 "Android SDK Platform 34"
   - 如果没有，勾选并点击 `Apply` 安装

2. **检查 JDK 版本**
   - `File` → `Project Structure` → `SDK Location`
   - 确保使用的是 JDK 17

3. **完全重新安装 Android SDK Platform 34**
   - 在 SDK Manager 中，取消勾选 "Android SDK Platform 34"
   - 点击 `Apply` 卸载
   - 重新勾选并安装
